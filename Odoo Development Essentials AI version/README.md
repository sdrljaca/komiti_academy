# Odoo Development Essentials AI Version

Ово је оригиналан, краћи AI водич инспирисан темама из књиге `Odoo 15 Development Essentials`, али није преписана нити препакована верзија те књиге.

Циљ овог директоријума је:
- да један кандидат или junior developer добије краћи и практичнији пут кроз Odoo development;
- да садржај буде ажуриран за Odoo 19;
- да објашњења буду ближа стварном раду у KomITi окружењу.

Ова верзија је намјерно краћа од књиге и организована као сет радних поглавља.

Ипак, то не значи да треба да буде само списак појмова. Зато је ова верзија писана тако да студент може:
- да чита поглавље као увод у тему;
- да из текста стекне mental model, а не само ријечник термина;
- да послије читања зна шта треба пробати у runtime-у;
- да лакше пређе на званичну Odoo 19 документацију.

## Како да читаш ово

- Не читај као теорију, него као радни handbook.
- Послије сваког поглавља уради бар једну малу практичну вјежбу.
- Ако нешто не можеш својим ријечима објаснити, рачунај да није још усвојено.
- За званичну и најтачнију reference документацију увијек провјери Odoo 19 docs.

Препоручени начин учења је овакав:

1. прво прочитај цијело поглавље без журбе;
2. затим подвучи или запиши нове појмове;
3. онда отвори свој локални Odoo и потражи те појмове у стварном систему;
4. тек послије тога иди на званичну документацију, да оно што читаш тамо не буде апстрактно.

Ово је важно зато што Odoo development брзо постане збуњујућ ако само читаш теорију без runtime слике, или ако само кликћеш по интерфејсу без јасног mental model-а.

## Шта је ново у односу на Odoo 15 фокус

Ова AI верзија има експлицитан Odoo 19 угао:
- упућује на Odoo 19 docs и Odoo 19 web framework;
- front-end дио је више усмјерен на Owl и registries/services/hooks;
- јаче наглашава testing, upgrade-safe thinking и runtime verification;
- уводи KomITi delivery дисциплину као саставни дио developer рада, не као посебну тему.

Другим ријечима: овај материјал не третира Odoo developer-а као особу која само пише Python. Циљ је да студент разумије цијели пут промјене, од model-а и view-а до upgrade-а, smoke test-а и runtime verification-а.

## Шта овај сет не покушава да буде

- није потпуна замјена за званичну документацију;
- није reference за сваки API;
- није копија туђе књиге;
- није довољан сам по себи без практичног рада у модулу и runtime-у.

## Минимални исход

Ако прођеш овај сет како треба, требало би да можеш:
- да поставиш Odoo development окружење;
- да направиш мали модул од нуле;
- да сигурно мијењаш постојећи модул;
- да разумијеш основни Odoo 19 frontend слој;
- да урадиш upgrade, smoke test и основну runtime верификацију;
- да разликујеш `код постоји` од `функционалност стварно ради`.

Ако овај сет добро послужи, студент на крају не би требало да има осјећај да је прочитао „AI скрипту“, него да је добио убрзан, разумљив увод у стварни Odoo development.
